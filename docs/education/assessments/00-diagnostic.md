# 진단 평가

## 목적

교육 시작 전 현재 이해 수준을 점검한다.

## 질문

1. 정적 분석은 왜 동적 import에 약한가
2. parser와 linker를 분리해야 하는 이유는 무엇인가
3. node와 edge의 차이는 무엇인가
4. `api-call-site`를 node로 두는 이점은 무엇인가
5. `spring-endpoint`와 `spring-controller`를 분리하는 이유는 무엇인가
6. `db-table`이 synthetic node라는 말은 무슨 뜻인가
7. impact 분석은 왜 reverse traversal이 필요한가
8. cold/warm 결과가 다르면 왜 심각한가
9. 지원 패턴 문서가 왜 필요하며, 왜 그것만 믿으면 안 되는가
10. 폐쇄망 반입 시 코드 외에 추가로 필요한 것은 무엇인가

## 채점 기준

- 0-3개: 기초 개념부터 시작
- 4-6개: Golden Path 중심 학습 필요
- 7-8개: 레이어별 분석으로 진입 가능
- 9-10개: 재설계 중심 학습 가능

## 현재 사용자 기준 해석

이번 교육의 시작점은 사실상 0점 가정으로 잡는다. 따라서 이 평가지는 기록용이며, 압박용 도구가 아니다.
