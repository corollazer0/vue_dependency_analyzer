# 최종 평가

## 목적

학습자가 실제로 이 시스템을 다시 설계하고 운영할 수 있는 수준에 도달했는지 평가한다.

## 평가 축

### 1. 구두 설명

다음 항목을 화이트보드 없이 10분 안에 설명한다.

- 전체 파이프라인
- Golden Path 2개
- parser/linker/analyzer 경계
- cache correctness 중요성

### 2. 문서 평가

아래 문서가 있어야 한다.

- Golden Path 추적서
- 지원 패턴 매트릭스
- 영향도 분석 요약
- 폐쇄망 운영 설계
- zero-base 재설계 문서

### 3. 실무형 문제 해결

다음 시나리오에 답한다.

- 특정 DTO 필드가 변경되면 어디까지 영향을 보는가
- MyBatis XML table 추출이 누락되면 어떤 운영 리스크가 생기는가
- monorepo/MSA 환경에서는 어떤 설정을 우선 설계해야 하는가
- 인증과 감사가 없는 상태로 왜 금융권 납품이 안 되는가

## 합격 기준

- 전체 파이프라인을 막힘 없이 설명한다.
- Golden Path 6개 중 최소 4개를 문서로 제출한다.
- supported/unsupported pattern 경계를 말할 수 있다.
- 폐쇄망 운영 설계를 최소 1차 수준으로 제시한다.
- zero-base 재설계 문서에 contract와 단계별 로드맵이 포함된다.
