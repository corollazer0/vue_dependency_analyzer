# 모듈별 점검표

## Module 01

- 정적 분석, AST, 그래프, parser/linker/analyzer 차이를 설명할 수 있다.
- cold/warm/incremental 차이를 설명할 수 있다.

## Module 02

- 주요 NodeKind와 EdgeKind를 분류할 수 있다.
- reverse adjacency와 file index 필요성을 설명할 수 있다.

## Module 03

- parse pipeline을 단계별로 설명할 수 있다.
- cache correctness 위험을 설명할 수 있다.

## Module 04

- Vue parser와 TS parser 역할을 구분할 수 있다.
- 프론트 지원/미지원 패턴 10개 이상을 설명할 수 있다.

## Module 05

- endpoint, DTO, DI, MyBatis statement, db-table 생성 방식을 설명할 수 있다.
- 백엔드 지원/미지원 패턴 10개 이상을 설명할 수 있다.

## Module 06

- unresolved edge가 어떻게 해소되는지 설명할 수 있다.
- API linker, MyBatis linker, DTO flow를 구분할 수 있다.

## Module 07

- circular, impact, DTO consistency, rules의 입력과 출력을 설명할 수 있다.
- change impact와 graph impact의 차이를 설명할 수 있다.

## Module 08

- CLI, server, UI 각자의 역할을 설명할 수 있다.
- 폐쇄망 운영 시 추가 요구사항을 설명할 수 있다.

## Module 09

- zero-base 재설계 원칙을 문서화할 수 있다.
- MVP와 제품형 설계를 분리할 수 있다.
