# 교육 프로그램 개요

## 1. 교육 목표

이 교육의 목표는 다음 한 문장으로 정리된다.

`이 저장소를 설명할 수 있는 상태`가 아니라 `같은 등급의 시스템을 다시 설계하고 구현할 수 있는 상태`를 만든다.

여기서 "이해"의 기준은 다음 5가지를 모두 만족해야 한다.

1. 전체 흐름을 말로 설명할 수 있다.
2. 주요 노드와 엣지가 어디서 생성되는지 말할 수 있다.
3. 어떤 경우에 분석이 실패하거나 약해지는지 말할 수 있다.
4. 결과를 어디까지 신뢰할 수 있는지 말할 수 있다.
5. 설계 대안을 제시할 수 있다.

## 2. 왜 이 프로젝트가 어렵나

이 시스템은 단순한 웹앱이 아니라 다음 성격을 동시에 가진다.

- 컴파일러 스타일 시스템: 소스를 읽고 구조화한다.
- 그래프 시스템: 노드와 엣지를 저장하고 탐색한다.
- 분석 시스템: 영향도, 순환, 정합성, 규칙 위반을 계산한다.
- 제품 시스템: CLI, 서버 API, UI, 캐시, watch mode를 제공한다.
- 운영 시스템: 사내 반입, 재분석, 배포, 추적성까지 고려해야 한다.

즉, 한 레이어만 이해해서는 전체를 다시 만들 수 없다.

## 3. 권장 학습 전략

가장 좋은 순서는 `기초 개념 -> Golden Path -> 레이어별 확장 -> 리스크/한계 -> 재설계`다.

사용자님의 원래 아이디어인 `vertical -> horizontal`은 방향은 맞다. 다만 바로 vertical trace부터 들어가면 개념이 없어서 파일 나열만 보게 된다. 따라서 아래 순서를 따른다.

1. 기초 개념
2. Golden Path 6개 추적
3. 레이어별 수평 정리
4. 실패 케이스와 미지원 패턴 정리
5. 폐쇄망 운영 관점 재정의
6. zero-base 재설계

## 4. 완료 기준

아래 질문에 막힘 없이 대답할 수 있으면 완료다.

- 이 시스템의 입력과 출력은 무엇인가
- 파서와 링커는 왜 분리되어 있는가
- 어떤 패턴은 감지되고 어떤 패턴은 감지되지 않는가
- `impact`, `dto consistency`, `rule violations`는 어떤 데이터 위에서 계산되는가
- `warm cache`와 `cold run`에서 결과가 왜 같아야 하는가
- 폐쇄망 반입 시 코드 외에 무엇을 추가해야 하는가

## 5. 학습 단계

### 단계 A. 기초 정렬

- 정적 분석
- AST
- 그래프
- 의존성 방향
- 정적 링크와 의미적 링크

### 단계 B. 시스템 추적

- Vue 화면에서 DB 테이블까지 내려가는 흐름
- DB 테이블에서 Vue 화면까지 올라오는 흐름
- 변경 파일에서 영향도 계산으로 가는 흐름

### 단계 C. 레이어별 분해

- graph model
- parsers
- linkers
- analyzers
- runtime surfaces
- operations

### 단계 D. 재설계

- 어떤 계약은 유지할지
- 무엇을 단순화할지
- 무엇을 강화할지

## 6. 교육 종료 시 기대 역량

- 아키텍트 관점: 시스템 경계와 품질 기준을 정한다.
- 리드 엔지니어 관점: 구현 순서와 팀 작업 단위를 나눈다.
- 운영 관점: 폐쇄망에서 설치, 분석, 배포, 추적 체계를 설계한다.
- 검증 관점: 지원 패턴, 미지원 패턴, 회귀 테스트 범위를 정의한다.

## 7. 금지할 학습 방식

- 파일을 순서 없이 전부 읽는 방식
- 동작을 확인하지 않고 문서만 읽는 방식
- 기능만 보고 운영을 무시하는 방식
- 코드 흐름은 모르는데 UI만 이해했다고 착각하는 방식
- 미지원 패턴을 모른 채 "완전 분석"이라고 가정하는 방식
