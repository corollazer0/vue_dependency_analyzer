# Lab 04. 폐쇄망 운영 설계

## 목적

이 실습의 목표는 "코드를 이해했다"를 "운영 가능한 설계로 바꿨다"로 끌어올리는 것이다.

## 전제 질문

- 사내망에 npm 외부 접근이 차단되어 있다면 어떻게 설치할 것인가
- 저장소 분석 이력은 어디에 남길 것인가
- 누가 어떤 프로젝트를 분석했는지 어떻게 추적할 것인가
- 결과를 누가 조회할 수 있어야 하는가

## 작성해야 할 항목

### 1. 배포

- 오프라인 패키지 방식
- 버전 관리 방식
- 업그레이드 전략

### 2. 보안

- 인증 방식
- 권한 모델
- 감사 로그
- 비밀값 관리

### 3. 운영

- 분석 작업 스케줄링
- 취소 및 재시도
- 백업/복구
- 모니터링

### 4. 품질

- cache correctness 검증
- fixture 기반 회귀 테스트
- supported pattern versioning

## 제출 산출물

- 폐쇄망 운영 설계서 1부
- 도입 체크리스트 1부

## 통과 기준

- 기술 구성뿐 아니라 운영 프로세스가 포함되어 있다.
- 인증/권한/감사 항목이 빠지지 않았다.
- 오프라인 빌드와 패키징 전략이 포함되어 있다.
